# Contributing

Thank you for your interest in contributing to this project.

## Before You Start

Please:

* Read the README
* Search existing issues before creating a new one
* Open an issue to discuss significant changes before starting implementation

## Reporting Bugs

When reporting a bug, please include:

* Steps to reproduce
* Expected behavior
* Actual behavior
* Environment details (OS, browser, version)
* Screenshots or logs if available

Use the provided Bug Report template whenever possible.

## Suggesting Features

Feature requests are welcome.

Please describe:

* The problem you are trying to solve
* The proposed solution
* Alternative approaches considered
* Expected benefits

Use the provided Feature Request template whenever possible.

## Development Setup

1. Fork the repository
2. Create a feature branch

```bash
git checkout -b feature/my-feature
```

3. Make your changes
4. Run tests and quality checks
5. Commit your changes
6. Open a Pull Request

## Pull Request Guidelines

Please ensure that:

* Code follows the existing project style
* New functionality includes tests where appropriate
* Existing tests continue to pass
* Documentation is updated when necessary
* Pull Requests remain focused on a single change

## Commit Messages

Use clear and descriptive commit messages.

Examples:

```text
feat: add thai translation support
fix: correct login redirect
docs: update installation guide
test: add API integration tests
```

## Code Review

All contributions are reviewed before being merged.

Feedback is intended to improve code quality and maintainability. Please be respectful and constructive during discussions.

## Questions

If you are unsure about a change, open an issue before investing significant development effort.

Thank you for contributing.
